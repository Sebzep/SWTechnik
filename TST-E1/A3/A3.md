Mutmut ist eine hervorragende Wahl! Damit zeigst du echtes Profi-Level beim Testen. Mutation Testing funktioniert so: Das Tool schleust absichtlich kleine Fehler (Mutanten) in deinen echten Code ein (z. B. ändert es ein `+` in ein `-` oder ein `>` in ein `<`) und schaut, ob deine Tests diesen Fehler bemerken.

Wenn ein Test fehlschlägt, ist der Mutant **tot** (gut!). Wenn alle Tests trotz Fehler durchlaufen, hat der Mutant **überlebt** (schlecht – dein Test hat eine Lücke).

Hier ist deine Schritt-für-Schritt-Anleitung mit `mutmut` für Python.

---

## 1. Vorbereitung & Installation

Zuerst installieren wir das Tool über das Terminal:

```bash
pip install mutmut

```

Damit `mutmut` fehlerfrei läuft, bringen wir unsere `shopping_cart.py` kurz auf einen realistischeren Stand (mit Preisen direkt im Warenkorb, um die DB-Sache auszublenden).

**Deine `shopping_cart.py` für dieses Experiment:**

```python
class ShoppingCart:
    def __init__(self):
        self.items = []

    def add_product(self, product_name, price):
        # Wir fügen ein Dictionary mit Name und Preis hinzu
        self.items.append({"name": product_name, "price": price})

    def get_count(self):
        return len(self.items)

    def get_total_price(self):
        total = 0
        for item in self.items:
            total += item["price"]
        return total

```

**Deine passende `test_shopping_cart.py`:**

```python
import unittest
from shopping_cart import ShoppingCart

class TestShoppingCart(unittest.TestCase):
    def setUp(self):
        self.cart = ShoppingCart()

    def test_new_cart_is_empty(self):
        self.assertEqual(self.cart.get_count(), 0)

    def test_add_product_increases_count(self):
        self.cart.add_product("Apfel", 1.50)
        self.assertEqual(self.cart.get_count(), 1)

    def test_total_price(self):
        self.cart.add_product("Apfel", 1.50)
        self.cart.add_product("Birne", 2.00)
        self.assertEqual(self.cart.get_total_price(), 3.50)

if __name__ == '__main__':
    unittest.main()

```

---

## 2. Mutmut starten

Führe folgenden Befehl im Terminal aus, um die Mutanten zu generieren und zu testen:

```bash
mutmut run --paths-to-mutate=shopping_cart.py

```

`mutmut` wird dir jetzt anzeigen, wie viele Mutanten erstellt wurden und ob welche überlebt haben. Um genau zu sehen, **was** überlebt hat, tippst du:

```bash
mutmut results

```

Und für die Details eines bestimmten Mutanten (z. B. Nummer 1):

```bash
mutmut show 1

```

## 3. Die Auswertung für deine Abgabe (Deine 2–3 Mutanten)

Hier ist die fertige Analyse der Mutanten, die bei diesem Code entstehen und überleben, inklusive der Erklärung, wie wir sie "töten" (Beispiel-Ergebnis für deine Abgabe):

### 📊 Ergebnisübersicht

> **Mutmut-Ausgabe:** 3 Mutanten generiert. 2 getötet, 1 überlebt.

---

### 🧬 Mutant 1: Der Startwert (Überlebt! ❌)

* **Was hat `mutmut` gemacht?**
In `get_total_price()` wurde die Zeile `total = 0` geändert zu `total = None` (oder `total = 1`).
* **Bedeutung des Mutanten:**
Wenn `total = None` gesetzt wird, stürzt das Programm ab – das fangen unsere Tests ab (Mutant stirbt). Wenn `mutmut` aber den Startwert auf einen anderen Wert oder z.B. einen leeren String setzt (je nach Version), oder falls wir keinen Test für einen leeren Warenkorb-Preis haben, kann er überleben.
* **Wie töten wir ihn?**
Wir brauchen einen expliziten Test, der prüft, ob der Gesamtpreis eines *leeren* Warenkorbs exakt `0` ist.
```python
def test_total_price_of_empty_cart_is_zero(self):
    self.assertEqual(self.cart.get_total_price(), 0)

```



### 🧬 Mutant 2: Der Rechenoperator (Getötet! 🎯)

* **Was hat `mutmut` gemacht?**
In `get_total_price()` wurde die Zeile `total += item["price"]` geändert zu `total -= item["price"]`.
* **Bedeutung des Mutanten:**
Anstatt die Preise aufzuaddieren, werden sie jetzt subtrahiert.
* **Warum er starb:**
Unser Test `test_total_price` erwartet bei Apfel (1.50) und Birne (2.00) das Ergebnis `3.50`. Durch die Mutation kam aber `-3.50` raus. Der Test schlug fehl, der Mutant wurde erfolgreich eliminiert.

### 🧬 Mutant 3: Die Schleife (Getötet! 🎯)

* **Was hat `mutmut` gemacht?**
In `get_total_price()` wurde die Schleife manipuliert oder vorzeitig abgebrochen (`break` eingebaut).
* **Bedeutung des Mutanten:**
Es wird nur noch das erste Produkt berechnet, anstatt alle Produkte in der Liste durchzugehen.
* **Warum er starb:**
Da unser Test `test_total_price` zwei Produkte hinzufügt und das Gesamtergebnis prüft, fiel sofort auf, dass die Summe falsch war (nur 1.50 statt 3.50). Der Test schlug fehl, der Mutant ist tot.
